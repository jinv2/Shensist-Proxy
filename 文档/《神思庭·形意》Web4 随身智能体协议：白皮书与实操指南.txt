# 《神思庭·形意》Web4 随身智能体协议：白皮书与实操指南

## 第一部分：理论基础 —— 纯粹的协议，绝对的主权

在传统的 Web2/Web3 架构中，用户必须依赖平台或复杂的本地部署。而《神思庭·形意》协议（以下简称“形意协议”）打破了这一桎梏，其核心理念可以浓缩为十二个字：**本地零运算，云端零存储，自由进出。**

1. **运算彻底外包 (API 路由化)：** 智能体的“听（STT）”、“看（Vision）”、“想（LLM）”、“说（TTS & Talking Photo）”全部剥离本地。本地设备（手机或 Ubuntu 电脑）仅仅作为一个**触发器**和**API 路由器**，直接将原始音视频流甩给顶级大模型 API。
2. **拒绝中心化部署 (P2P 意图网络)：**
没有“神思庭官方服务器”。节点与节点之间（Agent to Agent）采用去中心化点对点（P2P）协议通信。每一个使用该协议的人，都相当于掌握了自己数字分身的私钥。
3. **数据主权隔离 (灵魂不可篡改)：**
用户的《山海经》底蕴、人生阅历、谈判底线、核心利益，全部以纯文本形式留在用户本地设备中。每次调用大模型 API 时，这些信息仅作为 System Prompt（系统提示词）随用随发，阅后即焚，绝不留在任何云端。

## 第二部分：系统构成要素 (四大极简模块)

既然无需开发底层模型，形意协议的工程本质就是建立四个“标准化插槽”：

1. **【灵魂插槽】本地记忆与权限库：** 存储用户的姓名、底层价值观、谈判底线文本，以及公钥/私钥对（用于 P2P 通信寻址）。
2. **【算力插槽】多模态大模型 API：** 用户自行填入大模型服务商的 API Key（如 Gemini Pro 多模态版），为智能体接入“大脑”与“五官”。
3. **【表达插槽】拟真皮囊 API (可选)：** 用户填入音色克隆和照片驱动 API（如 ElevenLabs / D-ID），让智能体拥有主人的音容笑貌。
4. **【神经插槽】P2P 通信协议：** 一套极其轻量的 Python/JS 脚本，负责将本地的“意图（Intent）”打包加密，发送给另一个同样运行该协议的节点。

## 第三部分：极速制作与使用教程

由于没有任何复杂的环境部署要求，任何人只要有一台能联网的设备，就可以在 5 分钟内“捏”出自己的全权代表，并接入网络。

### 阶段一：初始化数字分身 (只需一次设置)

对于新用户，软件界面仅仅是一个极简的配置表单：

1. **注入灵魂：** 在“我是谁”文本框内，用自然语言写下你的人设和谈判底线（例如：*“我是神思庭，精通传统文化。我的底线是绝不接受 30% 以下的利润分成。如果对方施压，用《山海经》的隐喻迂回反驳。”*）
2. **挂载器官：** 在 API 填入框内，粘贴你申请到的大模型 API Key。
3. **生成凭证：** 软件自动在本地生成一段唯一的字符串（公钥）。**这就是你在这个网络里的“微信号”**。

### 阶段二：建立连接与授权 (自由进出)

1. **添加好友：** 你向对方索要他的“公钥字符串”，填入你的软件。两个智能体在底层完成了 P2P 握手，无需经过任何服务器。
2. **下达任务 (人类操作)：** 你在极简界面上输入今天的目标：*“去和节点 B 谈一下明天的合作日程，尽量把会议安排在下午。”*
3. **开启全权代理：** 点击【授权出击】按钮。此时，你可以把手机放回口袋，或者去喝茶。

### 阶段三：黑盒博弈 (API 自动互调)

在此刻，人类界面的下方只有简单的状态滚动条，而在网络底层，两个大模型正在以光速交锋：

1. **意图发送：** 你的节点将你的目标 + 你的本地底线，打包发给你的大模型 API。大模型生成第一轮谈判策略，加密后通过 P2P 协议发给节点 B。
2. **意图解析：** 节点 B 收到加密包，解密后连同 B 主人的底线，发给 B 的大模型 API 进行评估和反击。
3. **高速迭代：** 双方智能体（A 与 B）在几秒钟内完成几十轮来回拉扯、试探、妥协。所有的视觉识别（如果调用了摄像头）和语音合成，全由大模型 API 在云端瞬间完成。

### 阶段四：收网与人类裁决

1. **触发阈值：** 当双方大模型 API 判断已经达成共识，或者触碰到了无法逾越的“本地死线”时，自动停止 P2P 通信。
2. **一键确认：** 你的手机界面弹出绿色的摘要卡片：*“已达成共识，会议定在下午3点，对方态度积极。”* 下方是【同意】与【拒绝】按钮。人类按下按钮，一锤定音。
